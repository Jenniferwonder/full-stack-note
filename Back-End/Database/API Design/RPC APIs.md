# RPC APIs

- Has Endpoints for each action
- GET: for read only request
- POST: for the rest

### Pros

- Easy to understand
- Lightweight payloads
- High performance

###  Cons

- Discovery is difficult
- Limited standardization
- Leads to function explosion
